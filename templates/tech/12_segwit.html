Segregated Witness (aka. Segwit) essentially splits up transactions so only the data necessary for sending a transaction is in the part of a block that every node is guaranteed to read and understand. The data used for making sure the transaction is authorized (ie. the witness) is moved into a separate structure. The two biggest effects of this are:

  1. Blocks can effectively be twice as large without causing a network split (ie. those who disagree with Segwit do not need to run a Segwit node or accept Segwit transactions)
  2. This process also fixes "transaction malleability" which essentially means that it's a lot safer to build many layer 2 networks on top of Bitcoin. Prior to this fix it was much harder because people could produce two valid transactions for a layer 2 network that both involved the same BTC. If this wasn't fixed people would be able to double spend their BTC on networks like Lightning.
